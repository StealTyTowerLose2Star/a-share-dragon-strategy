# A 股龙头战法策略系统

专业的 A 股龙头股识别与跟踪系统，基于市场共识和游资战法逻辑，自动识别主线板块和龙头股，提供科学的买卖点建议。

## 🎯 核心功能

### 1. 主线板块识别

基于市场共识的板块周期判断标准：

| 周期阶段 | 连续强势天数 | 板块涨幅 | 涨停股数量 | 操作策略 |
|----------|--------------|----------|------------|----------|
| **启动期** | 1-2 日 | 前 10 | 2-3 只 | 轻仓试错 |
| **发酵期** | 3-5 日 | 前 5 | 5-10 只 | 加仓主线 |
| **高潮期** | 6-10 日 | 前 3 | 10+ 只 | 持股待涨 |
| **退潮期** | 热度下降 2 日 + | 跌出前 20 | <2 只 | 清仓观望 |

**主线确认标准**：
- 连续 **3 日** 板块涨幅前 10 → 确认为主线
- 连续 **2 日** 跌出前 20 → 退潮信号

### 2. 龙头股识别模型

#### 核心特征（按重要性排序）

1. **连板高度**（权重 40%）
   - 板块内连板数最高
   - 至少 3 连板以上才算龙头
   - 5 连板以上是总龙头级别

2. **涨停时间**（权重 25%）
   - 板块内最先涨停（9:30-9:35 最佳）
   - 封单金额大（>1 亿）

3. **带动效应**（权重 20%）
   - 龙头涨停后，板块跟风股上涨
   - 龙头分歧日，板块不崩

4. **抗跌性**（权重 15%）
   - 板块调整日，龙头横盘或微跌
   - 板块反弹日，龙头先涨

#### 龙头等级划分

| 等级 | 连板数 | 板块地位 | 操作策略 |
|------|--------|----------|----------|
| **总龙头** | 7+ 板 | 市场标杆 | 持有到断板 |
| **主线龙头** | 5-6 板 | 主线核心 | 持有到 5 日线 |
| **板块龙头** | 3-4 板 | 板块领涨 | 持有到 3 日线 |
| **补涨龙** | 1-2 板 | 龙头调整后 | 快进快出 |

### 3. 综合评分模型

```python
龙头评分 = (
    板块地位分 * 35 +      # 板块内排名
    连板高度分 * 25 +      # 连板数
    涨停时间分 * 20 +      # 涨停早晚
    带动效应分 * 15 +      # 跟风股数量
    抗跌性分 * 5           # 调整日表现
)
```

**龙头等级判定**：
- 综合评分 >= 80 分 → 总龙头（7 板+）
- 综合评分 60-79 分 → 主线龙头（5-6 板）
- 综合评分 40-59 分 → 板块龙头（3-4 板）
- 综合评分 < 40 分 → 补涨龙（1-2 板）

### 4. 自动报告生成

系统自动生成每日报告，包含：
- 主线板块分析
- 龙头股跟踪
- 新主线观察
- 风险提示

## 💰 仓位定位

**本系统定位**：总仓位的 **20%（1/5）** 用于龙头战法/打板策略

**剩余 80%（4/5）仓位**：配置趋势策略（趋势轮动 + 趋势跟踪）

| 策略类型 | 仓位占比 | 风险等级 | 持有周期 | 预期收益 |
|----------|----------|----------|----------|----------|
| **龙头战法** | 20% | 高 | 3-10 天 | 50-100% |
| **趋势轮动** | 40-50% | 中 | 1-3 月 | 20-40% |
| **趋势跟踪** | 30-40% | 中 | 2 周 -2 月 | 15-30% |

**后续规划**：我们将创建独立的「趋势策略系统」，包含趋势轮动和趋势跟踪两个模块，用于管理剩余 80% 仓位

## 📈 策略逻辑详解

### 主线板块判断流程

```python
def judge_sector_stage(sector_data):
    """
    判断板块周期阶段
    """
    consecutive_days = sector_data['连续强势天数']  # 连续前 10 天数
    hot_limit_up = sector_data['涨停股数量']
    volume_change = sector_data['成交量变化']
    
    if consecutive_days >= 6 and hot_limit_up >= 10:
        return "高潮期"
    elif consecutive_days >= 3 and hot_limit_up >= 5:
        return "发酵期"
    elif consecutive_days >= 2 and hot_limit_up >= 2:
        return "启动期"
    else:
        return "观察期"
```

### 龙头股识别流程

```python
def identify_leader(stocks_in_sector):
    """
    识别板块龙头
    """
    scores = []
    for stock in stocks_in_sector:
        score = (
            stock['连板数'] * 40 +
            (10 - stock['涨停时间_rank']) * 2.5 +
            stock['带动效应分'] * 20 +
            stock['抗跌性分'] * 15
        )
        scores.append((stock, score))
    
    return max(scores, key=lambda x: x[1])[0]
```

### 必要条件与充分条件

**必要条件**（必须全部满足）：
1. 连板数 >= 3
2. 板块连续 2 日前 10（主线确认）
3. 板块内连板数最高

**充分条件**（满足越多越强）：
1. 涨停时间 < 9:35（强）
2. 封单金额 > 1 亿（强）
3. 带动效应 > 3 只跟风（强）
4. 抗跌性：板块跌它不跌（强）

## ⚠️ 风险提示

1. **数据延迟**：免费数据源可能有 15 分钟延迟
2. **市场风险**：极端行情下策略可能失效
3. **个体差异**：推荐股票需结合个人风险偏好
4. **仅供参考**：不构成投资建议，盈亏自负

### 风控建议（针对 20% 龙头战法仓位）

**单只股票**：≤ 龙头战法等仓位的 50%（即总仓位的 10%）

**止损规则**：
- 跌破 5 日线减仓
- 跌破 10 日线清仓
- 主线退潮信号出现时及时止盈

**退潮期**：保持空仓或轻仓（≤ 龙头战法等仓位的 20%）

**总体原则**：龙头战法亏损不影响 80% 趋势策略仓位
